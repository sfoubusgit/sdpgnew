This document describes the complete architecture, purpose, and behavior of the Guided Prompt Interview Creator, a standalone application designed to build extremely high-quality Stable Diffusion prompts through an adaptive question-and-answer interview system.

============================================================
1. PURPOSE OF THE APPLICATION
============================================================

The Guided Prompt Interview Creator is an interactive system that helps users build complex and nuanced Stable Diffusion prompts without manually constructing text, understanding AI prompt engineering, or navigating complex settings.

Instead of presenting long lists of tags, toggles, and text inputs, the app works like a creative assistant that asks the user structured questions. Each answer leads to the next, eventually producing a rich, highly detailed, weight-enhanced AI image prompt.

This makes the tool ideal for:

Beginners who want guidance

Artists who want structured work

Professionals who want fine-grained control

Users who want high-quality, consistent output

The application focuses exclusively on Guided Interview Mode.
Other features such as randomization, preset builders, tag editing, or library managers are intentionally not included.

============================================================
2. TOP-LEVEL CONCEPT
============================================================

The application behaves like a branching interview system, where each question leads to the next based on the user's answers.

Each question can include:

A main question ("What hair type?")

Answer choices

Optional refinement questions

Optional weight sliders

Optional descriptions

Optional auto-suggest buttons

Each question is part of a decision tree, defined entirely through JSON files.

Whenever the user provides an answer, the system:

Records the answer

Moves to the next appropriate node

Updates the prompt preview

Displays additional refinements or weight controls when necessary

This continues until the interview reaches a natural conclusion.

============================================================
5. DECISION TREE SYSTEM
============================================================

All flow logic is defined in JSON files stored in:

src/data/interview/


Each file defines one or more nodes in the decision tree.

Nodes contain:

id

question

description (optional)

answers (required)

refinements (optional)

weights (optional)

File loading

The application automatically loads all .json files in that folder using:

import.meta.glob('/src/data/interview/*.json', { eager: true })


This means:

Adding new interview categories is as simple as creating new JSON files

No code changes are required

============================================================
6. INTERVIEW ENGINE
============================================================

The core logic is implemented in a React hook:

useInterviewEngine()


Responsibilities:

6.1 Track State

Current question node

History of visited nodes

Selected answers

Selected refinements

Slider values for weighted attributes

6.2 Navigation

next(answerId) moves to the next node

previous() goes back to the prior node

reset() restarts the interview

6.3 Real-Time Preview

getPreviewPrompt() returns the partial or final prompt text, dynamically assembled.

6.4 Completion

isFinished() identifies when no further nodes remain.

The engine creates a structured pipeline from the user‚Äôs responses to the final prompt.

============================================================
7. WEIGHT SYSTEM (ADVANCED NUANCE)
============================================================

Any node or refinement node may include weight sliders.

Weights adjust how strongly something is represented in the final prompt:

Example:

"weights": [
  {
    "id": "curl_intensity",
    "label": "Curl Intensity",
    "template": "curl intensity",
    "min": 1.0,
    "max": 1.3,
    "step": 0.01,
    "default": 1.12
  }
]


This creates a slider.

The final prompt will then include:

(curl intensity:1.12)


Weights allow for:

nuance

subtlety

intensity control

stylization

depth

This system is essential for high-quality prompt engineering.

============================================================
8. PROMPT ASSEMBLY
============================================================

A dedicated file:

src/services/promptAssembler.ts


Collects:

The main subject

All selected attributes

All refinement answers

All weights

Then it outputs a professionally structured Stable Diffusion prompt.

Example output:

A young adult female character with defined curly hair (curl intensity:1.18),
standing in an enchanted forest (magical atmosphere:1.32),
anime style, soft ambient lighting, subtle particle effects.

Negative prompt:
deformed, distorted, extra limbs, low detail, low quality, bad anatomy


The preview panel updates continuously as the user builds their character or environment.
